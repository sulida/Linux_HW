1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

git checkout .    - в файлах
git checkout <file_name>   - в конкретном фалй

2.Как вернуть файл из индекса в рабочий каталог?

git restore --staged 

3.Как отменить ошибочный commit ?

 git reset
git reset HEAD - вернуться назад
            ^ - на 1 коммит
            ^^ - на 2 коммита
или
            ~1 - на 1 коммит


4.Как правильно исправить ошибочный commit после отправки на gitHub?
git commit --amend
git push --force   - желательно только в своем репозитории применять эту команду

Если репозиторий рабочий, лучше снова сделать 
git add
git commit
git push

Будут видны коммиты и изменения


5.Как создать новую ветку и переключится в нее?
git branch
git checkout

или

git checkout -b <название ветки>

6.В чем отличение между fast-forward merge  and non fast-forfard merge?
Fast-forward merge - слияние, когда ветка, в которую происходит слияние , не имеет новых commit после создания ветки, из которой происходит слияние.
Non fast-forfard merge -слияние, когда ветка уже имеет commits после создания ветки, которая будет сливаться. 

7.Когда может возникнуть конфликт слияния веток, как  его разрешить?

Когда вносяться изменения из двух веток в один  и тот же файл или в часть кода.
Состояние можно проверить командой
git status

Войти в файл в редакторе и оставить или удалить те изменения, которые нужны.
Далее добавить файл и commit
git add
git commit -m ""

8.Можно ли удалить ветку? Как? 
Можно, но при удалении нужно выйти из ветки, которую хотим удалить:

git checkout ДРУГАЯ_ВЕТКА - переключиться на другую ветку
git branch -d ВЕТКА_НА_УДАЛЕНИЕ - удалить ветку
